# セッション進捗の記録

`progress-summary.md`（プロジェクトルートに存在しない場合は作成）を最新状態に更新してください。

## 手順

1. まず現在の `progress-summary.md` を読み込む（存在する場合）
2. このセッションの会話履歴を振り返り、以下を把握する：
   - 今日の日付（currentDate から取得）
   - 今日実施した作業内容（実装・修正・調査・テストなど）
   - 解決した問題と、その解決方法
   - 未解決の問題・課題
   - 次回以降のネクストアクション（優先度つき）
3. `progress-summary.md` を以下のフォーマットで上書き保存する

## 出力フォーマット

```markdown
# 進捗サマリー

## 最終更新：YYYY-MM-DD

---

## ✅ 完了した作業

### YYYY-MM-DD（直近セッション）
- （箇条書きで実施内容を記録。何を・なぜ・どう解決したかを簡潔に）

### YYYY-MM-DD（前回以前のセッション）
- （過去の履歴があれば残す）

---

## 🔧 技術的な決定事項・注意点

- （「なぜそのアプローチを選んだか」「ハマったポイント」など、次回再開時に役立つ情報）

---

## 📋 残課題・ネクストアクション

### 優先度：高（次回セッションで必ず着手）
- [ ] （タスク名）: （何をすべきか具体的に）

### 優先度：中
- [ ] （タスク名）: （概要）

### 優先度：低
- [ ] （タスク名）: （概要）

---

## 🗂 プロジェクト構成メモ

（ファイル構成・環境変数・接続先など、次回再開時に確認が必要な情報）
```

## 注意事項

- 過去のセッション記録は**削除せず残す**（日付つきで履歴として蓄積する）
- 技術的な決定事項は理由とともに記録する（「なぜそうしたか」が重要）
- ネクストアクションは次回セッション開始時にそのまま着手できる粒度で書く
- 環境変数・エンドポイント・リソース名など「また調べるはめになる情報」は必ず記録する
